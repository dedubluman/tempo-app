=== DESIGN DECISIONS ===

Dual-layer detection:
  1. WebSocket (fast): detects pending tx, sets receivedTxHash, shows "Payment detected!",
     plays sound. Fires before block confirmation.
  2. Balance polling (confirmation): existing useEffect watches for balance increase on
     each block, transitions to "success" state once on-chain confirmed.

TIP-20 calldata parsing (no viem ABI decode needed):
  - tx.to == expectedToken (token contract address)
  - input[0..10) == transfer/transferWithMemo selector (computed via toFunctionSelector)
  - input[34..74) == recipient address (last 40 chars of 64-char padded field)
  - input[74..138) == amount as uint256 (accept if txAmount >= expectedAmount)

Fallback trigger: WS connect timeout 5s, onerror, or unexpected onclose all start polling.
Polling dedup: lastProcessedBlockNum prevents re-checking same block on repeated ticks.
